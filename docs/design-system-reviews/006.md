## Key process insight — review + implement in a single cycle

Iters 01–05 filed beads but **implementation lagged review**. As of iter-05 closeout, 36 open `ds-review` beads had accumulated, including iter-00 P1s still unfixed 5 iterations later. The user's `/loop` intent explicitly called this out ("have a separate agent implement the fixes; even actual temperature") — keep review and implementation in lockstep so the backlog doesn't grow unbounded.

Iter-006 validated this pattern:
- Review phase: 1 agent, ~2 min, 6 findings
- Triage phase: parent session, skill patch + 5 bead files
- Implementation phase: 1 agent, ~45 min, 8 beads closed, 5 commits
- Audit phase: parent session, this doc + final commit

The implementation agent deliberately included **5 iter-00 P1 beads** in its fix scope because their files were currently modified in the batch. Fixing while the context was warm was cheaper than re-opening those files later. Net: the 8 closures included both new-in-this-iteration beads AND long-standing debt on the same file surface.
