## Skill patch — R14 promotion

Added to `SKILL.md` §2 hard rules table:

> **R14**: Every interactive element (`<button>`, `<a>`, `<input>`, custom widgets with `tabindex`) MUST have a `:focus-visible` rule in scoped CSS. Canonical: `outline: var(--border-width-thick) solid var(--color-focus); outline-offset: 2px`. Zero-`:focus-visible` selectors is a WCAG 2.4.7 fail — UA defaults are inconsistent (often absent on dark surfaces, often weak on coloured buttons).

## Key insight — Panel `role="dialog"` → `<aside>` was a semantic upgrade beyond the literal fix

`Codex-qz4p` asked for `aria-modal` correction on Panel's `role="dialog"`. The implementation agent instead flipped the element to `<aside>` (native landmark → implicit `role="complementary"`) with `aria-labelledby` pointing to an `sr-only` dynamic label that tracks the current level. Screen readers now announce "Brand editor — Colors" / "Brand editor — Brand Editor" as a complementary region, not as a blocking dialog.

This is the *correct* semantic — the brand editor panel is a persistent toolbox, not a modal dialog. Bolting on `aria-modal="false"` would have been the literal fix but left the wrong role. The agent's judgment call delivered a structurally better outcome.

**Lesson for future implementation briefs**: describe the *intent* (a persistent complementary tool) rather than the *literal fix* (add `aria-modal="false"`). The agent can produce better outcomes when it understands the semantic target.
