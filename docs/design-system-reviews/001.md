## Process learnings (fed into §7 of SKILL.md)

- Batch size of 7 files × one review agent yielded 22 findings in ~5 min of agent time. Good density.
- Severity triage matrix (block/major/minor/nitpick) works — the nitpicks really would have polluted beads if they'd been filed.
- Having the reviewer cite rule numbers (R1, R4) + reference sections (04-motion.md §2) made findings trivially actionable. This format should be preserved verbatim for future iterations.
- Skill-content findings and code findings interleave naturally — keep both streams in the same review rather than splitting.
