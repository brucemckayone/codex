## Stacked bug narrative

The user said "two places can toggle light and dark; they're not synced". Iter-019 proves this is the surface symptom of **three independent design flaws**:

1. **Write contention** (9u8wg): two controls write the same DOM attribute with different persistence contracts.
2. **Read staleness** (micw3): the UI that DISPLAYS the state has no subscription to the source of truth.
3. **Capture-restore drift** (z91af): the editor treats the pre-session theme as immutable across the session, when it isn't.

Fixing any one leaves the other two latent. All three need beads. Landing all three yields a clean two-axis model:
- Axis A = user theme preference (owned by `theme.ts`; touched only by the sidebar toggle)
- Axis B = editor's "which dark-override palette am I editing" (owned by `brand-editor-store`; scoped to `.org-layout[data-editing-theme]`)

The axes become orthogonal — which is the semantic intent of the two controls.
