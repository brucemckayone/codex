## Skill patches (3 new §§ + 1 meta)

### Ref 05 §Media Elements §7 — `<track>` suppression justification

`svelte-ignore a11y_media_has_caption` without a comment is a hidden WCAG 1.2.2 violation. Suppression MUST be paired with one of:
- `<!-- Intro video is muted/decorative — no captions required -->`
- `<!-- Captions URL not yet wired — tracked in beads-<id> -->`

### Ref 05 §Media Elements §8 — CSS `@media` over JS `matchMedia` one-shot

`window.matchMedia('(prefers-reduced-motion: reduce)').matches` at mount is **non-reactive**. Users who toggle the setting mid-session are stuck with the mount-time value. Prefer CSS `@media` (auto-reactive). JS with `addEventListener('change')` subscription is acceptable when the value must be read in script for calculations.

### Ref 05 §Media Elements §9 — Portaled overlay keyboard scoping

Components using `use:portal` escape the wrapper DOM, so `wrapperEl?.contains(activeElement)` returns `false` even when the overlay is focused. Correct pattern: attach `onkeydown` to the portal node itself with `tabindex="-1"` and auto-focus on mount via `$effect`.

### Meta — shared composable insight

The review's finding #25 (extract `createMediaKeyboardHandler`) was the signal for the helper extraction. Worth naming as a loop pattern: when ≥3 files share an anti-pattern, the implementation agent should consider a shared helper rather than N inline fixes. Added inline to the impl brief for iter-012; future impl briefs should include this scope guidance.

## Key insight — MCP gate has a known limitation

Chrome DevTools MCP `emulate` doesn't expose `prefers-reduced-motion` toggle directly. Implementation agent verified the CSS `@media` rule presence via stylesheet enumeration (`CSSRule.MEDIA_RULE`) instead. This is a partial verification — compiles correctly, but doesn't prove the animations actually halt when the user opts in.

Worth documenting in ref 09 MCP verification: when toggling a user preference isn't achievable via `emulate`, stylesheet-rule-presence verification is acceptable but must be explicitly called out as such. Not filing a bead — belongs in the ref 09 patch for a future iteration.
