## Process learnings

- **Targeting under-exercised references is the highest-ROI batch strategy.** Iter-04 chose reference 10 deliberately; it produced 3× more skill patches than iter-01. Future iter rotations should continue prioritising under-exercised refs.
- **"Is the skill accurate?" is a first-class review question.** The Path C drift would not have been caught if the reviewer hadn't been told to verify it. Future iterations should name one reference to stress-test in the dispatch prompt.
- **Recurrence counts earn stronger phrasing.** The skill now cites iteration numbers when a pattern recurs (e.g., "Caught in iter-01, iter-03, iter-04"). This makes priority self-evident to future agents — 4-recurrence bugs outrank 1-recurrence ones.
- **R12 elevation justifies Beads as audit history.** Without beads-labelled throughput from iter-01 (Codex-lmex), iter-03, and iter-04, the 3rd-recurrence pattern wouldn't have been noticed. The beads archive IS the evidence for skill rule upgrades.
