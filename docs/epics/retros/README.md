# Epic Retros

One file per completed epic: `<epic-id>.md`, written by **`codex-epic-retro`** post-merge.

A retro records what actually happened (the bd-audit tally, the `codex-review` findings, the merged
PRs) and where each lesson hardened — `bd remember`, a new lint/task bead, an ADR draft, a SKILL.md
rule, or a CLAUDE.md proposal. Recurring patterns (3+ cycles) graduate to
`docs/crystalizations/ROLLING-INVENTORY.md`, which `codex-crystalize` then promotes up the ladder
(`docs/epics/conventions.md` §5).

These docs are the timestamped, append-only narrative of the loop's learning — the Codex-tailored
equivalent of nmemo's per-skill MEMORY.md, kept as git-tracked docs instead (per `bd prime`: no
per-skill MEMORY.md files).
