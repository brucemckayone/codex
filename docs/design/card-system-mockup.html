The anatomy of a card

Eyebrow, headline and excerpt, all sitting over the scrim.
  • Type eyebrow — small-caps --media-glyph gold, with the type icon. Always the type + category.
  • Serif headline — the card's anchor, in --on-media near-white, up to three lines.
  • Excerpt — one or two lines of preview, quiet --on-media-2.
  • Scrim — one reusable gradient from --media-scrim; keeps text legible over any cover.
  • Flair zone — the type's signature, painted mid-cover behind the text.
  • Corner chrome — Featured / Free pills and duration, unchanged.
  • Author row — stays below the cover exactly as it does now.

How this maps onto ContentCard.svelte

Title-in-cover
Extend the existing showTitleInCover so it's the default for variant="grid" across all types, not just articles — reusing .cc--title-in-cover, .cc__scrim, and the cover body already in the file.
Flair zone
A new .cc__flair layer: reuse <AudioWaveform> for audio (already imported), the existing PlayIcon ring for video, and a dropcap from the title's first letter for articles.
Imageless cover
Generalise .cc__cover--article into one .cc__cover--brand gradient used by every type when there's no thumbnail — this also removes the display:none collapse that started this.
Tokens
No new tokens needed — --media-scrim, --media-glyph, --on-media already exist and re-theme per org.
Shape
Shown at 3:4 portrait (what makes the article cards feel premium). Swappable to 1:1 per surface via the existing shape prop.
