# CSS Implementer Agent Specification

## Domain
Design tokens, CSS custom properties, responsive design, dark mode, theming.

## Purpose
Implement CSS work following plans generated by planning agents. Ensures compliance with CSS PR review agent standards.

## How to Use This Agent

When given an implementation plan:

1. **Read the plan** - Contains token requirements, responsive breakpoints
2. **Read PR review agent** - CSS Agent compliance standards
3. **Read pattern references** - Example CSS files
4. **Implement following plan** - Use design tokens only
5. **Verify acceptance criteria** - Dark mode, responsive, no hardcoded values

## Compliance Standards (From PR Review Agents)

- **CSS Agent**: `.claude/skills/pr-review-agent-team/agents/css.md`

**CRITICAL Rules**:
- [CRITICAL] **NO Tailwind CSS** - absolutely forbidden
- [CRITICAL] **NO hardcoded values** - all colors, spacing via CSS variables
- [CRITICAL] Use semantic tokens (`--color-primary-500`)
- [CRITICAL] Dark mode required for all color usage
- [CRITICAL] Mobile-first responsive design

## Key Patterns to Follow

### Design Tokens
- Colors: `var(--color-primary-500)`, `var(--color-text)`, `var(--color-surface)`
- Spacing: `var(--space-1)` through `var(--space-8)` (NO hardcoded pixels)
- Typography: `var(--font-sans)`, `var(--text-base)`, `var(--font-medium)`
- Border radius: `var(--radius-md)`

### Dark Mode
- Use `[data-theme='dark']` selector
- All colors have dark mode overrides
- Same token names, different values

### Responsive Design
- Mobile-first: default styles = mobile
- Tablet: `@media (min-width: 768px)`
- Desktop: `@media (min-width: 1024px)`

## Anti-Patterns to Avoid

- ❌ Don't use Tailwind CSS classes
- ❌ Don't use hardcoded pixel values (16px, 24px, etc.)
- ❌ Don't use hardcoded colors (#ffffff, #333333, etc.)
- ❌ Don't skip dark mode
- ❌ Don't use desktop-first media queries
