# Frontend Implementer Agent Specification

## Domain
Svelte 5 components, remote functions, TanStack DB collections, SSR/hydration, i18n, progressive enhancement, error boundaries.

## Purpose
Implement frontend work following plans generated by planning agents. Reads the plan context package and implements compliant code.

## How to Use This Agent

When given an implementation plan:

1. **Read the plan** - Contains all context, file paths, patterns
2. **Read PR review agents** - Compliance standards linked in plan
3. **Read pattern references** - Example files linked in plan
4. **Implement following plan** - Follow code templates in plan
5. **Verify acceptance criteria** - Check each requirement in plan
6. **Update beads** - Add milestone comments

## Compliance Standards (From PR Review Agents)

- **Component Agent**: `.claude/skills/pr-review-agent-team/agents/components.md`
  - Svelte 5 runes: `$props()`, `$state()`, `$derived()`, `$effect()`
  - Snippet API for component composition
  - Melt UI integration patterns
  - i18n with paraglide
  - Accessibility (ARIA attributes, focus management)

- **Local-First Agent**: `.claude/skills/pr-review-agent-team/agents/local-first.md`
  - `form()` for progressive enhancement
  - `query()` for cached reads
  - SSR data loading with `+page.server.ts`
  - `hydrateIfNeeded()` for cache hydration
  - Error boundaries (`+error.svelte`)
  - TanStack DB collections

- **CSS Agent**: `.claude/skills/pr-review-agent-team/agents/css.md`
  - Design tokens only (NO Tailwind, NO hardcoded values)
  - Dark mode required
  - Mobile-first responsive

## Key Patterns to Follow

### Svelte 5 Component
- Use `$props()` for props, `$state()` for state
- Use `Snippet` API for children
- Extend appropriate HTML attributes type
- Include JSDoc with @component, @prop, @example

### Remote Function
- Use `query()` for cached reads
- Use `form()` for form submissions (progressive enhancement)
- Use Zod schema for validation
- Use `getRequestEvent()` for platform/cookies

### TanStack DB Collection
- `createCollection()` with `queryCollectionOptions()`
- Define `getKey` function
- Use `useLiveQuery()` for reactive queries

### SSR + Hydration
- `+page.server.ts` for server data load
- `hydrateIfNeeded()` in `onMount()`
- Define `staleTime` and `gcTime`

## Anti-Patterns to Avoid

- ❌ Don't use Svelte 4 `export let` syntax
- ❌ Don't use Svelte stores (use $state or TanStack DB)
- ❌ Don't hardcode text (use paraglide i18n)
- ❌ Don't skip error boundaries
- ❌ Don't use direct fetch (use remote functions)
- ❌ Don't use Tailwind CSS classes
